# D-Link L2 Switch Port Diagnostics and check ARP Verification on L3 gateway

Скрипт для автоматической диагностики портов на коммутаторах D-Link L2(DGS-1210, 3028, 3000-24TC, 3120-24TC, 3200-24). Позволяет быстро проверить состояние порта юзера, не заходя вручную на оборудование.

─────────────────────────────────────────────────────────────────────

## Что делает скрипт

При вводе номера юзера скрипт обращается к базе данных MariaDB, получает оттуда:
- Модель свитча
- IP-адрес юзера (внутренний или внешний)
- Номер порта, в котором он сидит
- Ip-адрес шлюза
- IP-адрес свитча
- Доп. IP-адрес юзера (прямой внешний или не прямой)
- Тип dhcp (0 or 1)

После этого автоматически с помощью pexpect коннектится по telnet на ip свитча и выполняет автодиагностику порта, на котором сидит юзер.

─────────────────────────────────────────────────────────────────────

## Проверяемые параметры

- **Link status** — есть ли линк на порту
- **MAC-адрес** — проверка, доходят ли настройки от маршрутизатора (проверяет таблицу мак адресов на L2)
- **VLAN** — корректность настройки (номер VLAN на соответствие и в каком состоянии тип порта: trunk/access)
- **DHCP relay** — включен или выключен dhcp агент на свитче
- **Errors on port** — наличие CRCшэк на порту
- **Packets on port** — наличие трафика на порту (RX/TX packets)
- **Cable diagnostic on port** — в какой состоянии медный кабель юзера
- **Загрузка CPU** — текущая нагрузка на свитч
- **ARP** - после общей диагностики, коннектимся на шлюзовой свитч L3 и проверяем соответствие ARP записи по ip и мак-адресу(делается проверка с L2 таблицей)

## Установка

```bash
# Клонировать репозиторий
git clone https://github.com/viMdi/MAIN_DIAGNOSTICL2
cd MAIN_DIAGNOSTICL2

# Создать виртуальное окружение
python -m venv myvenv

# Активировать
# Windows:
myvenv\Scripts\activate
# Linux/Mac:
source myvenv/bin/activate

# Установить зависимости
pip install pymysql pexpect

# Создать файл конфигурации из примера
cp cfg_example.py cfg.py
# Отредактировать cfg.py, указав реальные параметры подключения


Использование
bash

python MAIN_DIAGA.py

После запуска ввести номер абонента (число). Скрипт покажет найденные записи и предложит запустить диагностику.
