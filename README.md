# D-Link L2 Switch Port Diagnostics with L3 ARP Verification

Скрипт для автоматической диагностики портов на коммутаторах D-Link L2 с проверкой ARP-записей на оборудовании L3. Позволяет быстро проверить состояние порта абонента, не заходя вручную на оборудование.

─────────────────────────────────────────────────────────────────────

## Что делает скрипт

При вводе номера абонента скрипт обращается к базе данных MariaDB, получает оттуда:
- IP-адрес абонента
- шлюз (gateway)
- IP-адрес коммутатора
- номер порта

После этого автоматически подключается по telnet к указанному коммутатору и выполняет диагностику порта, на котором сидит абонент.

─────────────────────────────────────────────────────────────────────

## Проверяемые параметры

- **Link status** — есть ли линк на порту
- **MAC-адреса** — проверка, доходят ли настройки от маршрутизатора (появление MAC-адреса и ARP-записи с L3 оборудования)
- **VLAN** — корректность настройки (номер VLAN, тип порта: trunk/access)
- **DHCP relay** — включен или выключен на коммутаторе
- **Ошибки на порту** — наличие CRC error и других проблем
- **Трафик** — активность на порту (RX/TX packets)
- **Диагностика кабеля** — состояние кабельной линии
- **Загрузка CPU** — текущая нагрузка на коммутатор

## Установка

```bash
# Клонировать репозиторий
git clone https://github.com/viMdi/MAIN_DIAGNOSTICL2
cd MAIN_DIAGNOSTICL2

# Создать виртуальное окружение
python -m venv myvenv

# Активировать
# Windows:
myvenv\Scripts\activate
# Linux/Mac:
source myvenv/bin/activate

# Установить зависимости
pip install pymysql pexpect

# Создать файл конфигурации из примера
cp cfg_example.py cfg.py
# Отредактировать cfg.py, указав реальные параметры подключения


Использование
bash

python MAIN_DIAGA.py

После запуска ввести номер абонента (число). Скрипт покажет найденные записи и предложит запустить диагностику.
